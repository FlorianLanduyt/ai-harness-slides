What is an AI Harness?

The runtime orchestration layer that wraps an LLM API, implementing the agentic loop, tool dispatch, and state management required for autonomous operation.

User Interface
CLI / Web UI / IDE Extension
↓ stdin / HTTP ↑ streaming chunks
AI Harness
query() Agentic Loop
Tool[] Tool Registry
Message[] Context Buffer
permissions Auth Layer
↓ messages + tools ↑ stream / tool_use
LLM API
Claude / GPT / Gemini (stateless completion endpoint)

Without Harness

response = llm.complete(prompt)
# Returns text. That's it.
# No memory, no actions, no tools.

With Harness

while (true) {
  response = llm.complete(messages, tools)
  if (response.tool_use) {
    result = executeTool(response.tool_use)
    messages.push(result)
    continue  // Loop back to LLM
  }
  return response  // Done
}

while (true) {
  messagesForQuery = await applyAutoCompact(messages)

  for await (const message of callModel(messagesForQuery)) {
    yield message  // Stream to UI
    if (hasToolUse(message)) needsFollowUp = true
  }

  if (needsFollowUp) {
    for await (const result of runTools(toolUseBlocks)) {
      messages.push(result)
    }
    continue  // Loop back
  }

  return { reason: 'end_turn' }  // Done!
}

04

Permission System

Defense in depth — multiple layers ensure safety with configurable permission modes.

default

Ask Mode

Prompts user for dangerous operations. Safe default for most users.

auto

YOLO Mode

Auto-approve with ML classifier. For experienced users who trust the system.

plan

Read-Only

No writes allowed. Perfect for exploration and planning phases.

bypass

Trust All

Allow everything. Only for fully trusted environments.

Context Management

Managing the ~200K token context window with auto-compaction and smart truncation.

System Prompt Instructions & capabilities
User Context CLAUDE.md, date, environment
System Context Git status, working directory
Conversation History Compacted when needed
Tool Results Truncated if too large

Auto-Compaction

When tokens exceed ~180K, old messages are summarized by a compaction agent

Microcompaction

Large tool outputs are saved to temp files, with truncated preview in context

Key Architectural Patterns

Battle-tested patterns from years of production AI system iteration.

🌊

Generator-Based Streaming

Async generators yield messages as they arrive. Consumer controls iteration with clean cancellation.

async function* query(): AsyncGenerator<Message>
🧵

Context Threading

ToolUseContext carries state accessors, abort handling, and options through the entire call stack.

tool.call(input, context, canUseTool, onProgress)
🔄

Recovery Patterns

Multi-level recovery for max_output_tokens, prompt_too_long, and rate_limit errors.

for (const strategy of recoveryStrategies)
🎛️

Feature Flags

Compile-time elimination of unused features. Gradual rollout with zero runtime cost.

const Tool = feature('FLAG') ? require('./Tool') : null
