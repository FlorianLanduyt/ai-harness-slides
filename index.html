The Core Query Loop

The heart of the agentic system — a while(true) loop that continues until the model stops requesting tools.

An async generator implementing the ReAct pattern: Reason → Act → Observe → Repeat. Returns AsyncGenerator<Message> for streaming.

async function* query(
  messages: Message[],
  options: QueryOptions
): AsyncGenerator<Message, QueryResult> {

  while (true) {
    // 1. Context compaction if approaching token limit (~180K threshold)
    const messagesForQuery = await applyAutoCompact(messages, {
      threshold: MAX_CONTEXT_TOKENS * 0.9,
      compactionAgent: summaryModel  // Separate LLM call for summarization
    })

    // 2. Stream response via async iteration (backpressure-aware)
    let toolUseBlocks: ToolUseBlock[] = []
    for await (const chunk of streamModel(messagesForQuery, tools)) {
      yield chunk  // Yield each SSE chunk to UI immediately

      if (chunk.type === 'content_block_start' && chunk.content_block.type === 'tool_use') {
        toolUseBlocks.push(chunk.content_block)
      }
    }

    // 3. No tools requested → conversation turn complete
    if (toolUseBlocks.length === 0) {
      return { reason: 'end_turn', tokenUsage }
    }

    // 4. Execute tools in parallel with permission checks
    const results = await Promise.all(
      toolUseBlocks.map(async (block) => {
        const tool = toolRegistry.get(block.name)
        const permitted = await checkPermission(tool, block.input)
        if (!permitted) return { type: 'tool_result', error: 'Permission denied' }
        return await tool.call(block.input, context)
      })
    )

    // 5. Append results and loop back to model
    messages.push(...results)
  }
}
Key Implementation Details:
  • AsyncGenerator enables backpressure — consumer controls iteration pace
  • yield streams each chunk; return provides final metadata
  • Tool execution is parallel via Promise.all but respects dependency ordering
  • Compaction uses a separate model call to summarize old messages
File: src/core/query.ts — The entire agentic behavior is ~200 lines. Recovery logic handles max_output_tokens, rate_limit, and prompt_too_long errors with exponential backoff.

interface Tool<TInput, TOutput> {
  // Identity & Discovery
  name: string                           // Unique identifier for tool_use blocks
  description: string                    // Shown to model in system prompt

  // Input Validation (Zod schema → JSON Schema for API)
  inputSchema: ZodSchema<TInput>

  // Permission & Execution
  isReadOnly(): boolean                  // false = requires write permission
  needsPermission(input: TInput): boolean  // Dynamic permission check

  call(
    input: TInput,
    context: ToolUseContext,            // Abort signal, progress callback, env
    canUseTool: CanUseTool,              // Nested tool permission checker
    onProgress?: (p: Progress) => void
  ): Promise<TOutput>

  // Output Transformation
  mapToResult(output: TOutput): ToolResult  // Format for model consumption

  // UI Rendering (React component)
  renderToolUse(input: TInput): ReactNode
  renderToolResult(result: TOutput): ReactNode
}

Defense in depth — multiple layers ensure safety with configurable permission modes.

Multi-layer permission checking: PermissionMode → tool.needsPermission() → canUseTool() hook → user confirmation UI.

type PermissionMode = 'default' | 'plan' | 'auto' | 'bypass'

type CanUseTool = (
  tool: Tool,
  input: unknown,
  context: ToolUseContext
) => Promise<'allow' | 'deny' | 'ask'>

// Permission check flow
async function checkPermission(tool, input, mode): Promise<boolean> {
  // Layer 1: Mode-level gates
  if (mode === 'bypass') return true
  if (mode === 'plan' && !tool.isReadOnly()) return false

  // Layer 2: Tool-level permission check
  if (!tool.needsPermission(input)) return true

  // Layer 3: Auto-mode ML classifier (trained on user patterns)
  if (mode === 'auto') {
    const prediction = await classifier.predict(tool, input)
    if (prediction.confidence > THRESHOLD) return prediction.allow
  }

  // Layer 4: User confirmation UI (blocks until response)
  return await showPermissionDialog(tool, input)
}
isReadOnly() FileReadTool: true, BashTool: false, GrepTool: true
needsPermission() Dynamic check — BashTool checks for rm, sudo, etc.
allowlist User-configured patterns: "allow npm test", "deny rm -rf"
session cache Once approved, similar operations auto-allowed for session

Token budget management via tiktoken. Auto-compaction at 90% capacity. Microcompaction for large tool outputs via temp file references.

const MAX_TOKENS = 200_000
const COMPACT_THRESHOLD = 0.9 * MAX_TOKENS  // ~180K
const MICROCOMPACT_THRESHOLD = 50_000       // Per-message limit

async function applyAutoCompact(messages: Message[]): Message[] {
  const tokenCount = countTokens(messages)  // via tiktoken

  if (tokenCount < COMPACT_THRESHOLD) return messages

  // Find split point: keep recent, summarize old
  const splitIdx = findCompactionPoint(messages, {
    preserveLastN: 10,           // Always keep last 10 messages
    preserveToolResults: true,   // Keep pending tool results
  })

  // Summarize old messages via dedicated LLM call
  const summary = await compactionAgent.summarize(messages.slice(0, splitIdx))

  return [
    { role: 'system', content: `[Compacted: ${summary}]` },
    ...messages.slice(splitIdx)
  ]
}

// Microcompaction: large tool outputs → file reference
function microcompact(result: ToolResult): ToolResult {
  if (countTokens(result) < MICROCOMPACT_THRESHOLD) return result

  const tempPath = writeTempFile(result.content)
  return {
    content: truncate(result.content, 1000) +
             `\n\n[Full output: ${tempPath}]`
  }
}
Token Counting: Uses tiktoken (OpenAI's tokenizer) with cl100k_base encoding. System prompt ~8K tokens, leaving ~192K for conversation. Images counted as fixed token cost per dimension.

🔄 Error Recovery

const strategies = [
  { error: 'rate_limit',
    action: exponentialBackoff },
  { error: 'max_output_tokens',
    action: continueGeneration },
  { error: 'prompt_too_long',
    action: forceCompaction },

for (const s of strategies) {
  if (s.error === err.type)
    return s.action(err)
}
