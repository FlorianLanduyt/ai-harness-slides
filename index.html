The Core Query Loop

The heart of the agentic system — a while(true) loop that continues until the model stops requesting tools.

💬

→
🤖

Model Call

🔍

→
⚡

Execute

Run tools in parallel

↺
while (true) {
  messagesForQuery = await applyAutoCompact(messages)

  for await (const message of callModel(messagesForQuery)) {
    yield message  // Stream to UI
    if (hasToolUse(message)) needsFollowUp = true
  }

  if (needsFollowUp) {
    for await (const result of runTools(toolUseBlocks)) {
      messages.push(result)
    }
    continue  // Loop back
  }

  return { reason: 'end_turn' }  // Done!
}

Permission System

Defense in depth — multiple layers ensure safety with configurable permission modes.

default

Ask Mode

Prompts user for dangerous operations. Safe default for most users.

auto

YOLO Mode

Auto-approve with ML classifier. For experienced users who trust the system.

plan

Read-Only

No writes allowed. Perfect for exploration and planning phases.

bypass

Trust All

Allow everything. Only for fully trusted environments.

Context Management

Managing the ~200K token context window with auto-compaction and smart truncation.

System Prompt Instructions & capabilities
User Context CLAUDE.md, date, environment
System Context Git status, working directory
Conversation History Compacted when needed
Tool Results Truncated if too large

Auto-Compaction

When tokens exceed ~180K, old messages are summarized by a compaction agent

Microcompaction

Large tool outputs are saved to temp files, with truncated preview in context
